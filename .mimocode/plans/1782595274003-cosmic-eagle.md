# Refactor: Swarm as Preset, Not Mode

## Problem

Swarm was added as a `Mode` (user intent), but it's architecturally a `DeliberationMode` (execution strategy). Solo/duo/trio/fusion/hive are presets, not modes. Swarm should be the same.

## Changes

### 1. Revert `Mode` type
**File:** `packages/chimera-core/src/types/agent.ts`
- Remove `'swarm'` from `Mode` type

### 2. Add swarm to `DeliberationMode`
**File:** `packages/chimera-core/src/coordinator/deliberation/types.ts`
- Add `'swarm'` to `DeliberationMode` union

### 3. Wire swarm into DeliberationEngine
**File:** `packages/chimera-core/src/coordinator/deliberation/engine.ts`
- Add `runSwarm()` method that delegates to `SwarmOrchestrator`
- Add swarm case to `run()` switch
- Add swarm to `buildDelegatedConfig` for auto mode

### 4. Remove swarm from session-orchestrator mode dispatch
**File:** `packages/chimera-core/src/session-orchestrator.ts`
- Remove `'swarm'` from `VALID_MODE_PRESET_COMBOS`
- Remove `executeSwarm()` and `decomposeForSwarm()` methods
- Remove swarm import
- Update `mapModeToDeliberationMode` — swarm is now a preset, not a mode

### 5. Update CLI
**File:** `packages/chimera-cli/src/cli-router.ts`
- Remove `swarm` command
- Swarm is now reachable via `chimera code --preset swarm "task"` or auto mode selection

### 6. Fix all Record<Mode> references
**Files:**
- `packages/chimera-core/src/prompts.ts` — remove swarm from MODE_INSTRUCTIONS
- `packages/chimera-core/src/skills/skill-bundles.ts` — remove swarm from SKILL_BUNDLES

### 7. Add swarm to DeliberationConfig types
**File:** `packages/chimera-core/src/coordinator/deliberation/types.ts`
- Add `SwarmDeliberationConfig` interface
- Add swarm to `DeliberationConfig` union
