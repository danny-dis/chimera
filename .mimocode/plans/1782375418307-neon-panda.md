### The Core Question: Can a Cheap Model Match a Frontier Model?

**Yes, with the right harness.** Research shows:

1. **Diverse model teams outperform homogeneous by 9-11%** (RECONCILE, ACL 2024)
2. **Execution-grounded verification = +28 pts on SWE-bench Lite** (AgentForge, 2026)
3. **Role-specific model allocation = 72.4% on SWE-bench 500** (Agyn, 2026)
4. **Context degradation starts at 40-60% of window** — cheap models degrade faster
5. **GPT-4.1 mini beats GPT-4o on many benchmarks** at 1/6 cost — small models are catching up

The harness gap between cheap and frontier models can be closed by:
- **Better context management** (cheap models have worse long-context performance)
- **Tool-use scaffolding** (structured outputs, schema enforcement, retry)
- **Verification loops** (cheap draft + frontier review is nearly as good as frontier-only)
- **Instruction engineering** (system prompts that compensate for weaker instruction following)

## Key Architectural Decisions

### 1. Context Management Strategy

**Decision:** Implement the 5-stage compaction pipeline (snip → microcompact → reactive-compact → context-collapse → autoCompact) as defined in `phase-1-design.md`.

**Rationale:** This is the single biggest lever for cheap model performance. Cheap models degrade faster at high context fill. The pipeline runs mostly for free (stages 1-4 are in-memory) and only calls an LLM for autoCompact at 80% fill.

**Files:**
- `chimera-context/src/compaction/pipeline.ts` (~120 LOC)
- `chimera-context/src/compaction/thresholds.ts`
- 5 stage files under `compaction/`

### 2. Handoff vs Compaction

**Decision:** Use handoff-over-compaction (Sourcegraph Amp pattern) — spawn a fresh agent with a structured key-value handover document instead of lossy summarization.

**Rationale:** Research shows summarization loses 63% of information. A compact key-value handoff (200-600 tokens) preserves exact file paths, line numbers, and decisions that cheap models need.

**Trigger:** 60-70% context fill (well before degradation accelerates).

### 3. Verification-First Pattern

**Decision:** After every file edit, automatically run the narrowest useful check (test, lint, type) before proceeding.

**Rationale:** Cheap models make more errors. An automatic verification loop catches errors before they compound. This is the single biggest quality improvement from Claude Code best practices.

### 4. Coordinator-as-Prompt

**Decision:** For delegation-heavy tasks, use a system-prompt-based coordinator (369 lines) instead of the full runtime coordinator.

**Rationale:** The full `CoordinatorEngine` runtime is overkill for most delegation. A prompt that tells the main model how to be a delegator is simpler, cheaper, and proven in production (Claude Code).

### 5. sideQuery for Classification

**Decision:** Route all classification/intent-detection/scoring tasks through a cheap `sideQuery` LLM call instead of heuristics.

**Rationale:** A Haiku-class model call costs ~$0.0001 and provides much better classification than keyword heuristics. This is already proven in Claude Code's production system.
