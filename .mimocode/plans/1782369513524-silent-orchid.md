# Plan: Preset/Mode Distinction & Switching in TUI

## Context

The TUI currently conflates two distinct concepts:
- **Mode** (ask, plan, code, debug, review) — the task intent / prompt strategy
- **Preset** (solo, duo, trio, merge, fusion) — the agent topology / deliberation strategy

Mode is displayed in a `ModeSelector` bar but is display-only (no keyboard switching). Presets are not exposed in the TUI at all — they're only configurable via `.chimera/config.yaml` booleans (`fusion_mode`/`merge_mode`) which are parsed but never consumed at runtime.

The goal: give the user a clear, visually distinct UI for both dimensions with keyboard-driven switching.

## Approach

Extend the existing TUI layout with a two-row selector bar (Mode | Preset) that uses Tab-based focus cycling and arrow-key navigation. Add a new `PresetSelector` component, make `ModeSelector` interactive, and wire `/preset` slash command.

## Files to Modify

### 1. `packages/chimera-core/src/index.ts` — Export DeliberationMode
- Add `DeliberationMode` to the type exports so the TUI can import it

### 2. `packages/chimera-tui/src/types.ts` — Extend TUIProps
- Add `preset?: DeliberationMode` prop
- Add `onPresetChange?: (preset: DeliberationMode) => void` callback

### 3. `packages/chimera-tui/src/tui.tsx` — Root component
- Add `preset` state alongside existing `mode` state
- Add `focusIndex` state: `0 = input`, `1 = mode selector`, `2 = preset selector`
- Register root-level `useInput` for Tab (cycle focus), Escape (return to input)
- Pass `focused` boolean to `ModeSelector`, `PresetSelector`, and `Input`
- Pass `onPresetChange` callback
- Render both selectors side-by-side in a new row, with labels showing which is focused

### 4. `packages/chimera-tui/src/components/mode-selector.tsx` — Make interactive
- Accept `focused?: boolean` prop
- When focused: respond to left/right arrows (navigate), Enter (select)
- Visual feedback: focused selector gets brighter border/color, unfocused dims
- Remove the underscore prefix from `onModeChange` (currently `_onModeChange`)

### 5. `packages/chimera-tui/src/components/preset-selector.tsx` — NEW
- `PresetSelector` component with same interaction pattern as ModeSelector
- Displays: solo, duo, trio, merge, fusion with icons and descriptions
- Accepts `currentPreset`, `onPresetChange`, `focused`, `compact` props
- Arrow-key navigation + Enter to select when focused

### 6. `packages/chimera-tui/src/components/input.tsx` — Add focused gate
- Accept `focused?: boolean` prop (default `true` for backward compat)
- When `focused === false`, skip all keystroke processing (return early from `useInput`)

### 7. `packages/chimera-tui/src/commands/commands.ts` — Add /preset command
- Add `getPreset()` / `setPreset()` to `CommandContext`
- Add `/preset <solo|duo|trio|merge|fusion>` command handler
- Update `/status` to show current preset
- Update `/help` text
- Add 'preset' to `ALL_COMMANDS` for autocomplete

### 8. `packages/chimera-cli/src/cli-router.ts` — Wire preset state
- Track `currentPreset` alongside `currentMode`
- Pass preset to TUI props and command context
- Handle `onPresetChange` callback to update state + TUI

## Visual Design

```
CHIMERA v0.0.1
┌─ Chat ──────────────────────────────────┐
│  ...                                     │
└──────────────────────────────────────────┘
 Mode: ⚡code ◈plan ◉debug ◎review    Preset: ▸solo duo trio
> _
```

- Active selector row gets `cyan` color + bold label
- Inactive row gets `dimColor` gray
- Current selection in each row highlighted with underline
- Focused selector shows `▸` cursor before the label

## Keyboard Model

| Key | Input focused | Mode focused | Preset focused |
|-----|--------------|--------------|----------------|
| Tab | → Mode | → Preset | → Input |
| Shift+Tab | → Preset | → Input | → Mode |
| ← → | (ignored) | Navigate modes | Navigate presets |
| Enter | Submit | Select mode | Select preset |
| Esc | (ignored) | → Input | → Input |
| Arrows/text | Normal input | (ignored) | (ignored) |
