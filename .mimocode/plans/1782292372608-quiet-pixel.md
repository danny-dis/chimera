# Plan: Scheduled Loops (Cron-like Execution)

## Context

The current loop system only supports iteration-based execution (run N times or until condition). The user wants time-based scheduling — "run this task every day at 2pm." No scheduling infrastructure exists in the codebase.

## Goal

Add a `SchedulerManager` that stores cron-like schedules, evaluates them, and triggers workflow runs at the right times. Schedules persist to disk and survive daemon restarts.

## Step 1: Define Schedule Types

**File**: `chimera-core/src/workflow/types.ts`

```typescript
export interface ScheduleEntry {
  id: string;
  name: string;
  /** Cron expression: "0 14 * * *" = every day at 2pm */
  cron: string;
  /** Task description to send to the workflow */
  task: string;
  /** Workflow name to run (default: built-in loop workflow) */
  workflowName?: string;
  /** Optional workflow definition (inline) */
  workflow?: WorkflowDefinition;
  /** Max iterations per run (for loop-based workflows) */
  maxIterations?: number;
  /** Enabled flag */
  enabled: boolean;
  /** When the schedule was created */
  createdAt: string;
  /** Last time the schedule was triggered */
  lastRunAt?: string;
  /** Last run's status */
  lastRunStatus?: 'success' | 'error' | 'cancelled';
}

export interface ScheduleConfig {
  schedules: ScheduleEntry[];
}
```

## Step 2: Implement SchedulerManager

**File**: `chimera-core/src/workflow/scheduler.ts` (NEW, ~200 lines)

Core class with:

### Storage
- `schedules: Map<string, ScheduleEntry>` in memory
- Persist to `.chimera/schedules.json` on every change
- Load from disk on construction

### Timer Loop
- `setInterval` every 60 seconds to evaluate all enabled schedules
- For each schedule: check if current time matches cron expression
- If match found: dispatch the workflow via `WorkflowDispatcher`
- Update `lastRunAt` and `lastRunStatus`

### Cron Evaluation (simple, no library)
Since we want zero new dependencies, implement a minimal cron matcher:
- Parse cron expression into `{ minute, hour, dayOfMonth, month, dayOfWeek }`
- Match against current time
- Support: `*` (any), exact values (`5`), ranges (`1-5`), steps (`*/5`)

### CRUD Methods
```typescript
addSchedule(entry: Omit<ScheduleEntry, 'id' | 'createdAt'>): ScheduleEntry
removeSchedule(id: string): boolean
updateSchedule(id: string, updates: Partial<ScheduleEntry>): ScheduleEntry | null
listSchedules(): ScheduleEntry[]
getSchedule(id: string): ScheduleEntry | null
enableSchedule(id: string): boolean
disableSchedule(id: string): boolean
```

### Integration
```typescript
constructor(dispatcher: WorkflowDispatcher, eventStream?: EventStream)
```
- Uses `WorkflowDispatcher.dispatch()` to trigger workflows
- Emits events when schedules trigger
