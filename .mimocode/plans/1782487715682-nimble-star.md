# Chimera TUI Improvement Plan

## Overview

Refactor `packages/chimera-tui/` into a production-ready, terminal-agnostic TUI with:
- Zen aesthetic (muted palette, clean lines)
- Chat as primary interface
- Collapsible sidebar (Ctrl+B)
- Syntax highlighting in code blocks
- Command system wired end-to-end
- Responsive resize handling
- Zero new runtime dependencies

---

## Phase 1: Theme System

**Create `src/theme.ts`** (~60 lines)

Centralized palette and tokens. All components will import from here instead of hardcoding colors.

```ts
export const zen = {
  // Base
  bg: 'black',
  fg: 'white',
  // Semantic
  accent: 'cyan',        // headings, links, active states
  muted: 'gray',         // borders, dim text, placeholders
  success: 'green',      // completed status, additions
  warning: 'yellow',     // cost warnings, command mode
  error: 'red',          // errors, deletions
  info: 'blue',          // info, agent planner role
  // Agent role colors
  role: { writer: 'green', reviewer: 'cyan', challenger: 'magenta', synthesizer: 'yellow', planner: 'blue', researcher: 'white', summarizer: 'gray' },
  // Borders
  border: 'gray',
  borderActive: 'cyan',
} as const;
```

Also export a `codeSyntax` map for syntax highlighting colors (Phase 2).

## Phase 2: Syntax Highlighting

**Create `src/syntax.ts`** (~120 lines)

Hand-rolled tokenizer for fenced code blocks. No dependencies.

- `tokenizeCode(code: string, lang: string): SyntaxToken[]`
- Supported languages: TypeScript/JavaScript, Python, Rust, Go, Shell
- Token types: `keyword`, `string`, `comment`, `number`, `function`, `type`, `plain`
- Each token carries a `color` string matching ink's color palette

**Modify `src/components/markdown.tsx`**

- Import `tokenizeCode` from `./syntax.js`
- Replace `code_block` rendering (lines 254-268): render each line through `tokenizeCode` if `lang` is provided
- Keep plain rendering for unknown languages
- Split into `src/components/markdown/tokenizer.ts` and `src/components/markdown/renderer.ts` to stay under 300 lines? No — only add ~20 lines to markdown.tsx, keeping it under 320. Not worth splitting yet.

## Phase 3: Layout Refactor

**Modify `src/hooks/use-window-size.ts`** — no changes needed, already complete

**Create `src/hooks/use-layout.ts`** (~40 lines)

Layout computation hook:
- Calls `useWindowSize()` internally
- Computes: chat width, sidebar width, sidebar visible (based on toggle state)
- Returns `{ width, height, chatWidth, sidebarWidth, sidebarVisible, isMinSize }`
- Min size check: if columns < 80 || rows < 24, set `isMinSize = true`

**Modify `src/tui.tsx`**

Major refactor of the root component:

```
BEFORE:                          AFTER:
┌──────────────────────┐        ┌──────────────────────────┐
│ StatusBar            │        │ StatusBar                │
├──────────┬───────────┤        ├──────────────────────────┤
│ Chat 65% │ Sidebar   │   →    │ Chat (full width)        │
│          │ 35%       │        │                          │
├──────────┤           │        │                          │
│ Input    │           │        ├──────────────────────────┤
└──────────┴───────────┘        │ Input                    │
                                └──────────────────────────┘
                                + Sidebar overlay (conditional)
```

Changes:
1. Import `useWindowSize`, create `useLayout` (or inline)
2. Add `sidebarVisible` state (default `false`)
3. Add `activeOverlay` state: `null | 'sessions' | 'diff' | 'agents' | 'events'`
4. `useInput` handler for Ctrl+B (toggle sidebar), Escape (close overlay), Ctrl+C (exit)
5. Chat takes `flexGrow={1}` — fills available space
6. Sidebar renders conditionally when `sidebarVisible` is true
7. Overlays render conditionally on top of chat area

Minimum size check: if `!isMinSize`, render a centered message "Terminal too small. Resize to at least 80x24."

## Phase 4: Command System Integration

**Create `src/hooks/use-commands.ts`** (~50 lines)

Bridge between input and command dispatcher:
- Builds `CommandContext` from current TUI state (mode, preset, costData, history, sessionId)
- Exposes `executeCommand(text: string): CommandResult`
- Manages command history array

**Modify `src/components/input.tsx`**

- Accept optional `onSlashCommand?: (cmd: string) => void` prop
- When input starts with `/` and Enter is pressed, call `onSlashCommand` instead of `onSubmit`
- Add autocomplete: when Tab is pressed and input starts with `/`, call `autocompleteCommand` and fill

**Modify `src/tui.tsx`**

- Create `useCommands` hook with current state
- Wire `Input.onSlashCommand` → `useCommands.executeCommand`
- On result: add output as system messages, handle `viewHint` (set overlay), handle `clearMessages`, handle `exit`

**Modify `src/tui.tsx`**

**Modify `src/components/status-bar.tsx`**

- Import `zen` theme from `./theme.js`
- Replace hardcoded colors with theme tokens
- Add sidebar indicator: show `[Ctrl+B]` hint or sidebar state icon

## Phase 6: Overlay Integration

**Modify `src/tui.tsx`**

Wire in existing unused components as conditional overlays:

- `activeOverlay === 'sessions'` → render `<SessionBrowser>` in a box over the chat area
- `activeOverlay === 'diff'` → render `<DiffViewer>` with `props.diffFiles`
- `activeOverlay === 'agents'` → render `<AgentDashboard>` (compact/full toggle)
- `activeOverlay === 'events'` → render `<EventLog>` (expanded)

Escape key closes any active overlay.
